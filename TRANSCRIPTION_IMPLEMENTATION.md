# Transcription Service Implementation

This document describes the complete implementation of the automatic transcription service for Recontext.online using Faster Whisper and RabbitMQ.

## Overview

The transcription service automatically transcribes audio tracks when LiveKit track recordings complete. The system uses RabbitMQ for asynchronous task processing and Faster Whisper (medium model) for speech recognition with phrase-level timestamps.

## Architecture

### Components

1. **Managing Portal (Go)**: Sends transcription tasks to RabbitMQ when track recordings complete
2. **RabbitMQ**: Message queue for async transcription tasks
3. **Transcription Service (Python)**: Consumes tasks, transcribes audio, stores results
4. **PostgreSQL**: Stores transcription phrases and status
5. **MinIO**: Stores audio files

### Workflow

```
LiveKit Track Recording Completes
         ↓
Managing Portal receives egress_ended webhook
         ↓
Check if it's a track egress (not room composite)
         ↓
Construct m3u8 playlist URL from file path
         ↓
Send message to RabbitMQ queue with playlist URL
         ↓
Transcription Service picks up message
         ↓
Download m3u8 playlist from MinIO
         ↓
Parse playlist and download all .ts segments
         ↓
Combine segments with FFmpeg into single audio
         ↓
Transcribe combined audio with Faster Whisper
         ↓
Store phrase-level transcriptions in database
         ↓
Mark track as ready and cleanup temp files
```

## Backend Changes (Go)

### 1. RabbitMQ Publisher Package

**File**: `pkg/rabbitmq/publisher.go`

**Purpose**: Publish transcription tasks to RabbitMQ queue

**Key Functions**:
- `NewPublisher()`: Initialize RabbitMQ connection and declare queue
- `PublishTranscriptionTask()`: Send transcription task message
- `Close()`: Close RabbitMQ connection

**Message Format**:
```json
{
  "track_id": "uuid",
  "user_id": "uuid",
  "audio_url": "http://minio:9000/recontext/<egress-id>/playlist.m3u8",
  "language": "",
  "token": ""
}
```

**Note**: The `audio_url` points to an m3u8 HLS playlist file generated by LiveKit egress.

### 2. Managing Portal Updates

**File**: `cmd/managing-portal/main.go`

**Changes**:
- Added `rabbitMQPublisher *rabbitmq.Publisher` to `ManagingPortal` struct
- Initialize RabbitMQ publisher in `NewManagingPortal()`
- Read configuration from environment variables:
  - `RABBITMQ_HOST`
  - `RABBITMQ_PORT`
  - `RABBITMQ_USER`
  - `RABBITMQ_PASSWORD`
  - `RABBITMQ_QUEUE`

**File**: `cmd/managing-portal/handlers_livekit.go`

**Function**: `handleEgressEnded()`

**New Logic**:
1. After updating egress recording status
2. Check if egress completed successfully (status != "failed")
3. Query database to see if egress is associated with a track
4. If yes:
   - **Construct m3u8 playlist URL**:
     - If filePath ends with `.m3u8`: use as-is
     - If filePath contains `/`: append `/playlist.m3u8`
     - Otherwise: use `<egress-id>/playlist.m3u8`
   - Get room SID from track
   - Get meeting from room name
   - Get user ID from meeting creator
   - Send transcription task to RabbitMQ with:
     - Track UUID
     - User UUID
     - M3U8 Playlist URL
     - Language (empty for auto-detect)

## Transcription Service (Python)

### Directory Structure

```
transcription-service/
├── main.py                 # Entry point
├── worker.py              # RabbitMQ consumer
├── transcriber.py         # Faster Whisper wrapper
├── database.py            # PostgreSQL operations
├── config.py              # Configuration management
├── requirements.txt       # Python dependencies
├── Dockerfile             # Docker container
├── .env.example           # Environment template
└── README.md              # Service documentation
```

### Core Files

#### `config.py`
Loads configuration from environment variables:
- RabbitMQ connection settings
- PostgreSQL connection settings
- Whisper model configuration
- MinIO storage settings

#### `database.py`
Database operations:
- `create_transcription_tables()`: Create schema if not exists
- `save_transcription_phrases()`: Store phrase-level transcriptions
- `update_transcription_status()`: Track job status
- `mark_track_ready()`: Update track_recordings table
- `get_transcription_phrases()`: Retrieve transcriptions

#### `transcriber.py`
Whisper integration with m3u8 support:
- `download_m3u8_and_combine()`: Download m3u8 playlist, fetch all segments, combine with FFmpeg
- `download_audio()`: Auto-detect m3u8 or direct file, download accordingly
- `transcribe_audio()`: Transcribe local audio file
- `transcribe_from_url()`: Download + transcribe + cleanup

**M3U8 Processing**:
1. Download playlist file from URL
2. Parse playlist to extract segment URLs (.ts files)
3. Download all segments sequentially
4. Create FFmpeg concat file
5. Combine segments using `ffmpeg -f concat -c copy`
6. Return path to combined audio file
7. Cleanup segments and temp files

#### `worker.py`
RabbitMQ consumer:
- `connect()`: Connect to RabbitMQ
- `process_message()`: Handle transcription task
- `start()`: Begin consuming messages

### Database Schema

#### `transcription_phrases` Table

| Column | Type | Description |
|--------|------|-------------|
| id | UUID | Primary key |
| track_id | UUID | Reference to track recording |
| user_id | UUID | Reference to user |
| phrase_index | INTEGER | Sequential phrase number (0-based) |
| start_time | NUMERIC(10,3) | Phrase start time in seconds |
| end_time | NUMERIC(10,3) | Phrase end time in seconds |
| text | TEXT | Transcribed text |
| confidence | NUMERIC(5,4) | Confidence score (avg log probability) |
| language | VARCHAR(10) | Detected language code |
| created_at | TIMESTAMP | Creation timestamp |
| updated_at | TIMESTAMP | Last update timestamp |

**Indexes**:
- `idx_transcription_phrases_track_id` on `track_id`
- `idx_transcription_phrases_user_id` on `user_id`
- `idx_transcription_phrases_start_time` on `start_time`

#### `transcription_status` Table

| Column | Type | Description |
|--------|------|-------------|
| id | UUID | Primary key |
| track_id | UUID | Reference to track recording (unique) |
| status | VARCHAR(50) | Status (pending, processing, completed, failed) |
| started_at | TIMESTAMP | Job start time |
| completed_at | TIMESTAMP | Job completion time |
| error_message | TEXT | Error details if failed |
| phrase_count | INTEGER | Number of phrases transcribed |
| total_duration | NUMERIC(10,3) | Total audio duration in seconds |

**Indexes**:
- `idx_transcription_status_track_id` on `track_id`

## Docker Integration

### Transcription Service Dockerfile

```dockerfile

# Install system dependencies
RUN apt-get update && apt-get install -y \
    ffmpeg \
    libpq-dev \
    gcc \
    && rm -rf /var/lib/apt/lists/*

# Copy and install Python dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application code
COPY . .

# Create directory for Whisper models
RUN mkdir -p /app/models

# Set environment
ENV PYTHONUNBUFFERED=1
ENV WHISPER_CACHE_DIR=/app/models

```

## Performance Considerations

### Whisper Model Selection

| Model | Size | VRAM | Relative Speed | Accuracy |
|-------|------|------|----------------|----------|
| tiny | 39M | ~1GB | 32x | Low |
| base | 74M | ~1GB | 16x | Fair |
| small | 244M | ~2GB | 6x | Good |
| medium | 769M | ~5GB | 2x | Very Good |
| large | 1550M | ~10GB | 1x | Best |

**Default**: Medium model provides good balance of speed and accuracy

### GPU Acceleration

For faster transcription, use GPU:

```yaml
# docker-compose.yml
transcription-service:
  environment:
    - WHISPER_DEVICE=cuda
    - WHISPER_COMPUTE_TYPE=float16
  deploy:
    resources:
      reservations:
        devices:
          - driver: nvidia
            count: 1
            capabilities: [gpu]
```

**Speedup**: 5-10x faster with GPU

### Resource Requirements

**CPU Mode (Default)**:
- RAM: 6-8GB
- CPU: 4+ cores
- Disk: 2GB for model cache

**GPU Mode**:
- RAM: 4GB
- VRAM: 6GB+
- GPU: NVIDIA with CUDA support

## Error Handling

### Transcription Failures

- Failed transcriptions are logged with full error details
- Status is updated to "failed" with error message
- Messages are NOT requeued to prevent infinite loops
- Database cleanup handles partial transcriptions

### RabbitMQ Connection Issues

- Managing portal logs warning but doesn't fail if RabbitMQ unavailable
- Transcription service will retry connection on startup
- Messages are persisted in RabbitMQ (durable queue)

### Database Issues

- Transcription service creates tables automatically on startup
- Failed database writes are logged but don't crash service
- Transactions ensure consistency

## Troubleshooting

### Model Download

**Issue**: First run downloads Whisper model (~1.5GB)

**Solution**: Ensure internet connection and disk space. Use volume to persist models.

### Memory Issues

**Issue**: Service crashes with OOM

**Solution**:
- Use smaller model (small or base)
- Increase Docker memory limit
- Use GPU mode (uses VRAM instead of RAM)

### Audio Download Fails

**Issue**: Cannot download from MinIO

**Solution**:
- Check STORAGE_BASE_URL is correct
- Verify MinIO is accessible from transcription service
- Check network connectivity

### No Tasks Received

**Issue**: Transcription service not receiving messages

**Solution**:
- Verify RabbitMQ is running
- Check queue name matches in both services
- Verify managing portal successfully sends messages
- Check RabbitMQ management UI for queue status
