🧩 Общая концепция Recontext.online

Recontext.online — это on-premise (или edge-cloud) платформа для:

автоматической обработки и анализа аудио- и видеозаписей (в том числе потоковых),

распознавания речи с диаризацией (определением говорящих) и тайм-кодами,

контекстного поиска по текстам, смысловым фрагментам и даже по смыслу запросов,

автоматической суммаризации и тегирования разговоров,

записи, хранения и анализа видеоконференций, включая интеграцию с Jitsi и внешними системами связи.

⚙️ Основные модули
1. Media Ingestion Layer

Принимает аудио/видео файлы или потоки (MP4, WAV, WebRTC, RTSP и др.).

Очередь сообщений (Kafka / NATS / RabbitMQ) для масштабируемой обработки.

Возможность подключения к звонковым системам (Asterisk, 3CX, Twilio, SIP-интерфейсы).

2. Speech Processing

Модель Whisper (GPU) — распознавание речи с поддержкой многих языков.

PyAnnote / NeMo / Silero — диаризация (идентификация говорящих).

Генерация транскрипта с тайм-кодами, сохранение в формате JSON+SRT.

3. Semantic Layer

Векторизация (BERT / InstructorXL / OpenAI Embeddings / local LLM).

Сохранение в Qdrant или Milvus для контекстного поиска по смыслу.

Поиск по фразам, темам, вопросам ("кто говорил о бюджете?", "что решили по проекту?").

4. Summarization & Insight Extraction

Автоматическая суммаризация по темам/спикерам.

Ключевые слова, теги, эмоциональный тон (sentiment analysis).

Подготовка “meeting minutes” — структурированных итогов.

5. Jitsi Integration

Возможность создать/присоединиться к конференции прямо из Recontext.

Запись и хранение каждой сессии (аудио/видео + стенограмма).

Опционально — запуск бота-рекордера в существующие звонки (Jibri-like behavior).

6. Storage Layer

MinIO или S3-совместимое хранилище.

PostgreSQL / ClickHouse для метаданных и аналитики.

Контейнеризация в Docker Compose или Kubernetes.

7. Integration Layer

REST / gRPC API для доступа к записям, транскриптам, поиску.

WebSocket API для потоковой обработки.

Интеграции:

Asterisk/3CX/Twilio — авто-запись звонков и отправка в анализ.

CRM (Bitrix24, amoCRM, Salesforce) — связывание транскриптов с клиентами.

MS Teams / Zoom / Telegram / WhatsApp — парсинг звонков через API-адаптеры.

🧠 Возможности пользователя

Загрузить аудио/видео или подключить звонок → получить автоматическую расшифровку.

Найти нужный фрагмент по фразе, контексту или смыслу.

Получить резюме разговора, выделение решений, задач, эмоций.

Создать видеоконференцию (на Jitsi) и автоматически сохранить её результаты.

Хранить записи в каталогах по проектам/клиентам.
