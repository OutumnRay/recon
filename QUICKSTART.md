# 🚀 Быстрый старт

## Сборка и запуск проекта

### Обычная сборка (рекомендуется)

```bash
./build.sh
```

**Что происходит:**
1. ✅ Версия автоматически увеличивается: `0.1.0+0` → `0.1.0+1`
2. ✅ Генерируются файлы версий для frontend и backend
3. ✅ Запускается `docker compose up --build`
4. ✅ Версия встраивается в Docker образы

### Изменение версии перед сборкой

```bash
# Для релиза новой версии (0.1.0 → 0.2.0)
./version-bump.sh minor

# Для исправления багов (0.1.0 → 0.1.1)
./version-bump.sh patch

# Затем запустите сборку
./build.sh
```

## 🎯 Основное

| Действие | Команда |
|----------|---------|
| Собрать проект | `./build.sh` |
| Увеличить минорную версию | `./version-bump.sh minor` |
| Увеличить патч версию | `./version-bump.sh patch` |
| Проверить текущую версию | `cat VERSION` |

## ⚠️ Важно

- Файл `VERSION` - единственный файл, который нужно коммитить
- Файлы `version.ts` и `version.go` генерируются автоматически
- Всегда используйте `./build.sh` для сборки проекта
